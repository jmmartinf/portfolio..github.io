An market share analysis with a Maptree diagram. You can find small pieces of data analysis process: Import, Tidy, Transform, Visualize, Model, Communicate.

Data source

Eurostat is one of the most important data sources that I know, many regional statistics departments are nothing more than a subsidiary of the European Statistical Office, where they replicate the data. Yes indeed, the data becomes redundant with some obsolete application for the publication of the information, therefore, it is very convenient to know the catalogue, both the analytical and the geographical one (it is certainly worth integrating the data related to geography as one more layer of information), it doesn't matter, "inner join"!

This example uses the "eurostat" library, for data collection, but any JSON client will do, so don't hesitate to use "jsonlite", it works the same.

Soon, it will appear the new version of web services... more work to do the same, not surprising! Meanwhile, here is an example of how to extract information selectively, without having to perform intense filtering tasks.

Air passenger transport between main airports in each reporting country and partner reporting countries

This data table shows movements in the airports of Europe (arrivals and departures), in number of flights and number of passengers in monthly, quarterly and annual periods.

Getting Data

This example filters the information to only the flights arriving at the Canary Islands airports that present non-regional traffic, the value of the information lies in determining the economic activity of a region that has the tourism sector as the main contributor to GDP.

One of the recurrent tasks with the data coming from EUROSTAT is the conversion of data type in date fields ("2022M06") is not a friendly format to process the information. Hence the value of the "eurotime2date" function, maybe that's why I use this library.
